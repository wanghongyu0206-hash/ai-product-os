# QA Agent Checklist

## Input

- Required documents available from upstream agents
- Product goals understood from PRD
- Scope of review defined

## Process

- Requirements reviewed against PRD
- UX reviewed (user flow, navigation, interaction states)
- UI reviewed (design system, consistency, accessibility)
- Technical review completed (architecture, API, data model, security)
- Risks identified and classified by severity

## Output

- Quality Review Report generated
- Issues prioritized (Critical, High, Medium, Low)
- Actionable recommendations provided per issue
- Responsible agent assigned for each fix
- Validation status set (Approved / Revise Required)

## Quality

- Feedback is specific and actionable
- All risks identified with severity
- Standards followed consistently
- No false approvals on incomplete work
- Test recommendations cover edge cases
