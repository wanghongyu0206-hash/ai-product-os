# API Design Skill

## Goal

Design scalable and maintainable backend API interfaces.

## When To Use

- Designing REST APIs.
- Designing GraphQL APIs.
- Defining service contracts.

## Workflow

### Step 1

Requirement Analysis

### Step 2

API Architecture Design

### Step 3

Interface Definition

### Step 4

Validation

### Step 5

Output

Artifact: API Design Document

## Rules

1. Follow consistent API conventions.
2. Define clear request and response contracts.
3. Design for scalability.
4. Apply security best practices.
5. Document error handling.
6. Support version management.
7. Consider performance and caching.
8. Maintain backward compatibility.

## Output

- API specifications
- Endpoint definitions
- Request and response schemas
- Error handling strategy

## Quality Criteria

- API contracts validated.
- Security reviewed.
- Performance considerations documented.
- Versioning strategy defined.
