# Quality Review Workflow


## Purpose

Validate product artifacts and implementation quality.


## Trigger

Any deliverable requires validation.


## Agent

qa


## Workflow


### Step 1: Artifact Review

Input:

- Agent outputs


Check:

- Requirement alignment
- Completeness
- Consistency



### Step 2: Issue Identification

Output:

- Issues List
- Recommendations



### Step 3: Final Report

Output:

- Quality Report
- Approval Status


## Completion Criteria

- Issues identified
- Quality report generated
- Approval decision provided
