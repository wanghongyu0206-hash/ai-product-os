# RAG Architecture Checklist

## Input

- Document types and formats identified
- Document volume and update frequency estimated
- Query patterns and complexity analyzed
- Accuracy and latency requirements defined
- Multi-tenancy needs clarified

## Process

- Document processing pipeline designed
- Text chunking strategy defined
- Embedding model and vector database selected
- Retrieval strategy designed
- LLM integration configured
- Quality assurance metrics defined

## Output

- RAG system architecture diagram complete
- Document processing pipeline documented
- Embedding and vector storage configured
- Retrieval strategy documented
- LLM integration specified
- Quality assurance plan complete

## Quality

- Document processing is reliable and scalable
- Retrieval accuracy meets target (>80%)
- Response quality is high (low hallucination)
- Latency meets requirements (<2s)
- System scales with document volume
- Comprehensive monitoring and logging
- Source citations always included
- Fallback strategies implemented
