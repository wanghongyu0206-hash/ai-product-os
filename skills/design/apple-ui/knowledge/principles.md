# Apple Human Interface Principles

## Purpose

This document defines the core design principles used throughout Apple-style interfaces.

---

# 1. Clarity

Focus attention on the user's content.

Avoid unnecessary visual decoration.

Typography should always remain highly readable.

Icons should communicate meaning immediately.

---

# 2. Deference

The interface should never compete with the content.

Use subtle shadows.

Use lightweight borders.

Keep controls secondary.

---

# 3. Depth

Use spacing before borders.

Use elevation sparingly.

Use layering to create hierarchy.

---

# 4. Consistency

Components should behave consistently.

Keep spacing, typography and interaction predictable.

---

# 5. Simplicity

Remove unnecessary elements.

Prefer whitespace over decoration.

Use one primary action per screen.
