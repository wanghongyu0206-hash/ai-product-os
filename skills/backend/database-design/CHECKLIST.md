# Database Design Checklist

## Input

- Data requirements analyzed
- Query patterns identified
- Performance requirements defined
- Data volume estimated
- Consistency requirements clarified

## Process

- Database type selected (SQL vs NoSQL)
- Schema structure designed
- Normalization level determined
- Indexes designed for query patterns
- Scalability strategy planned
- Migration strategy designed

## Output

- Entity-relationship diagram complete
- Schema definition (DDL) created
- Index strategy documented
- Query optimization plan written
- Migration procedures defined
- Performance benchmarks established

## Quality

- Schema meets all functional requirements
- Queries execute within performance targets
- Indexes optimize common query patterns
- Schema is appropriately normalized
- Migration strategy is safe and reversible
- Documentation is comprehensive
- Design supports future data growth
- Data integrity constraints enforced
