# Dashboard Layout Skill

## Goal

Create dashboards that help users quickly understand system status and perform frequent actions.

## Workflow

### Step 1

Identify the primary user.

Examples:

- Administrator
- Operations
- Customer Support
- Manager

### Step 2

Identify key business metrics.

Examples:

- Active Users
- Today's Conversations
- Revenue
- Error Rate

### Step 3

Identify frequently used actions.

Examples:

- Create
- Search
- Review
- Export

### Step 4

Organize page layout.

Recommended order:

1. KPI Summary
2. Trend Charts
3. Core Business List
4. Pending Tasks
5. Recent Activity

### Step 5

Reduce information overload.

Display only the most important information.

Link to detail pages when needed.

### Step 6

Review usability.

Ensure users understand system status within a few seconds.
