# System Design Checklist

## Input

- Functional requirements defined
- Non-functional requirements identified (performance, scalability, availability)
- Constraints understood (budget, timeline, team skills)
- Integration points mapped

## Process

- Architectural pattern selected
- System components designed
- Data flow patterns defined
- Communication patterns chosen (sync/async)
- Scalability strategy planned
- Security architecture designed

## Output

- System architecture diagram complete
- Component specifications documented
- Data flow diagrams created
- API specifications defined
- Deployment architecture planned
- Disaster recovery plan documented
- Architecture decision records written

## Quality

- Architecture meets all functional requirements
- Non-functional requirements addressed
- Clear component boundaries
- Well-defined communication patterns
- Security considerations included
- Comprehensive documentation
- Trade-offs documented and justified
- Design reviewed by stakeholders
