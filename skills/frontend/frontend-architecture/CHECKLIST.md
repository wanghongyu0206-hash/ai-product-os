# Frontend Architecture Checklist

## Input

- Project requirements identified
- Team size and structure understood
- Technology stack selected
- Feature complexity analyzed

## Process

- Folder structure designed
- State management architecture defined
- Data flow patterns established
- Module boundaries created
- Architecture decisions documented

## Output

- Folder structure diagram complete
- State management map created
- Data flow diagrams documented
- Module boundaries defined
- Architecture documentation complete

## Quality

- Structure is clear and navigable
- State management is predictable
- Data flow is unidirectional
- Modules are loosely coupled
- Easy to onboard new developers
- Scales with team growth
- Architecture decisions are documented
