# Frontend Review Checklist

## Input

- Code changes identified
- Coding standards available
- Accessibility requirements defined
- Performance benchmarks established
- Architecture guidelines accessible

## Process

- Code structure reviewed
- Component design evaluated
- Accessibility implementation checked
- Performance optimizations assessed
- Code quality analyzed
- Security practices verified

## Output

- Review report generated
- Issues documented with severity
- Recommendations provided
- Action items prioritized
- Code examples included

## Quality

- Accessibility standards fully checked
- Performance optimizations verified
- Security vulnerabilities identified
- Code quality issues documented
- Actionable recommendations provided
- Clear priority ranking established
- Positive patterns highlighted
